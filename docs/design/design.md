# デザインガイドライン

この文書は、佐藤医院Webサイトのデザインに一貫性を持たせるための共通ルールをまとめたものです。新しいページや部品を作るときは、この文書と `css/style.css` の既存実装を確認してください。

## 1. デザインの方針

- 清潔感、安心感、親しみやすさを大切にする
- 医療情報を読み取りやすく、迷わず目的の情報へ移動できる構成にする
- 装飾を増やしすぎず、余白と情報のまとまりで見やすさをつくる
- ページごとに独自の表現を追加する前に、既存の色、書体、部品を再利用する

## 2. カラー

色は原則として、`css/style.css` の `:root` に定義されたCSSカスタムプロパティを使用します。

| 用途 | 変数 | 色 |
| --- | --- | --- |
| メインカラー | `--color-main` | `#2f8f83` |
| メインカラー（濃色） | `--color-main-dark` | `#278478` |
| アクセントカラー | `--color-accent` | `#46aac8` |
| 本文 | `--color-text` | `#333333` |
| 補足文 | `--color-muted` | `#656565` |
| 罫線 | `--color-line` | `#dce8e7` |
| 基本背景 | `--color-bg` | `#ffffff` |
| ミント背景 | `--color-bg-mint` | `#eaf6f3` |
| 薄いグレー背景 | `--color-bg-soft` | `#f5f7f7` |

- メインカラーは、ボタン、リンク、見出しの補助要素などに使用する
- アクセントカラーは、特に注目させたい見出しや情報に限定して使用する
- 本文と背景には十分なコントラストを確保する
- 新しい色が必要な場合は、用途を明確にしたうえで `:root` に変数として追加する

## 3. タイポグラフィ

- 基本書体は `Noto Sans JP` とする
- フォールバックは `Yu Gothic`、`Hiragino Kaku Gothic ProN`、`Meiryo`、sans-serifの順とする
- 本文の基準サイズは `16px`、行間は `1.65` とする
- ページ内の見出し階層は `h1` → `h2` → `h3` の順を守り、見た目のためだけに階層を飛ばさない
- 太字は見出しや重要な情報に使い、本文全体を太字にしない
- 小さな補足文でも、原則として `13px` 未満にしない

## 4. レイアウトと余白

- 標準のコンテンツ最大幅は `1180px`（`--content-width`）とする
- 読み物などの狭いコンテンツ幅は `860px`（`--content-narrow`）とする
- ページ左右の余白は、原則として `.section-inner` または `.section-inner-narrow` で管理する
- 関連する情報は近づけ、異なる情報のまとまりには十分な余白を設ける
- 固定の高さは、画像比率やUI上の理由がある場合だけ使用する

## 5. UI部品

### ボタンとリンク

- 主要な操作にはメインカラーを使用し、ページ内で最も目立つ操作を明確にする
- ボタンの文言は「詳しく見る」だけに頼らず、移動先や操作内容が分かる表現にする
- テキストリンクは、色だけでなく下線や矢印などでもリンクであることが分かるようにする
- ホバー、フォーカス、押下時の状態を用意する

### カードと囲み

- 角丸は既存部品に合わせ、基本は `12px` とする
- カードの境界には `--color-line`、背景の切り替えには `--color-bg-mint` または `--color-bg-soft` を使用する
- 影は必要最小限にし、罫線や背景色で区切れる場合は使用しない

### セクション見出し

- 共通の見出しには `.section-heading` を使用する
- 見出しは中央揃えを基本とし、長文や記事ページでは左揃えを選択できる
- 日本語見出しと補助ラベルの表記順、余白、色をページ間で統一する

## 6. 画像とアイコン

- 医院やスタッフの写真は、明るく自然で安心感のあるものを選ぶ
- 画像を不自然に引き伸ばさず、必要に応じて `object-fit: cover` でトリミングする
- 内容を伝える画像には適切な代替テキストを設定する
- 装飾目的の画像は、支援技術で不要に読み上げられないようにする
- アイコンは同じ線幅、テイスト、サイズ感のものを使用する

## 7. レスポンシブ対応

- デスクトップだけでなく、タブレットとスマートフォンでも情報の優先順位が保たれるようにする
- 既存のブレークポイント `1199px` と `820px` を基準にする
- スマートフォンでは横スクロールを発生させない
- タップ可能な要素は十分な大きさと間隔を確保する
- 表は情報を省略せず、必要に応じて横スクロールやレイアウト変更で対応する

## 8. アクセシビリティ

- キーボードだけでもすべての操作を行えるようにする
- フォーカス表示を消さず、現在位置が視覚的に分かるようにする
- 色だけで状態や意味を伝えない
- フォーム部品にはラベルを関連付け、エラー内容を具体的に表示する
- アニメーションは控えめにし、`prefers-reduced-motion` の設定を尊重する

## 9. 更新時のルール

- 共通デザインの変更は、個別ページではなく共通CSSへ反映する
- 似た部品を新しく作る前に、既存クラスを再利用または拡張できないか確認する
- 新しいルールを採用した場合は、この文書にも追記する
- ルールと実装が異なる場合は、意図を確認したうえで文書または実装を更新し、差異を残さない
