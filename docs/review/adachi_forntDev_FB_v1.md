## 総評

初心者のバイブコーディングとしては上位。見た目の完成度が高く、目に見えない品質（読み上げ・キーボード操作への配慮）まで自発的に入っている。伸びしろは2つ——**「依頼書との答え合わせ」** と **「最終の壊れチェック」**。どちらも実務で一番効く。

## 🟢 よくできている（伸ばす）

- デザインが依頼のトーンに乗っている（白基調＋淡グリーン・写真多用・清潔感）。カンプ→実装の再現度も高い。
- 見えない品質が入っている：本文スキップリンク／フォーカス枠／メニュー開閉をスクリーンリーダーに伝えるaria／表の見出し設定／動きを減らす設定への対応。初心者でここは立派。
- スマホ対応の骨格（2段階の切替・ハンバーガー・画像可変）が組めている。

## 🔴 直そう（このまま院長に出すと事故る）

1. **お知らせの日付が全部「YYYY.MM.DD」のまま**（公開サイトで確認）。データには正しい日付が入っているのに、日付を整える処理が"仮置き文字"を返したまま（`js/main.js` の `formatDate`）。→ 「`news-data.js` の日付を `2025.10.01` の形にして画面に出して」とAIに直させる。
2. **「よくあるご質問」ページが無い**。依頼書は6ページ指定で、原稿にもFAQ本文（診療・予防接種・支払い・その他）がフルである。ナビにも無い。→ 1ページ追加＋ナビ／フッターに導線。※カンプの時点で抜けている＝**自分のカンプでなく"依頼書"で答え合わせを**。
3. **一番大きいヒーローの見出しが独自の文言**（「患者様一人ひとりに寄り添った診療／…共に歩む医療」）。依頼書は **「モットー『地域に根ざした医療』を目立たせて」** と明記しているのに、そのモットーが主役になっていない。医療サイトで一番目立つ言葉を勝手に作るのは、実務では一番まずい。→ ヒーローの主役コピーをモットー基準に。

## 🟡 もっと良くなる

4. **タブレット幅（約820〜1180px）でヘッダーが横にはみ出す**。ヘッダー内側だけ幅が1180px固定で、フッター等のような画面幅追従になっていない。→ フッターと同じ `min(1180px, calc(100% - 48px))` に。
5. **診療時間表の記号を原稿の ●／ から ○× に変え、凡例が無い**。記号だけだと高齢の患者さんが迷う。原稿表記に戻すか、凡例（○＝診療 ×＝休診）を添える。
6. **トップの電話番号がタップ発信できない**（アクセスページはできている）。一番見られるトップこそ `tel:` リンクに。
7. **原稿の本文が要約され、理念が7→6項目に減っている**（「ライフステージに応じた医療」が落ちている）。意味を変えていないのは good だが、原稿は「改変不可・追加は要連絡」。載せ切るか、縮めるなら院長に確認を。

## 💭 考えてみよう

8. 原稿を短くしたいと思ったとき——「勝手に変える」「全部載せる」「"こう縮めていいですか"と確認する」。院長が一番安心なのはどれ？（実務の"うまさ"は技術より、この一手にある）
9. **スタッフ写真の丸い切り抜きが、1枚ずつ手打ちの数字（`-174.499%` 等）で位置合わせ**されている。今は綺麗だが、写真を差し替えた瞬間に崩れる。→ これはAIが「とりあえず動く形」で作った典型。`object-fit: cover` ＋ `object-position` で同じことがずっと簡単・頑丈になる。"動くけど脆い"を見抜けると一段上。

## 根拠・確認箇所（どこを見てそう言っているか）

各指摘の裏取り。ファイル名・行と、依頼書／原稿のどこと照らしたかを載せる。自分でも同じ手順で確かめてみてください。

- **🔴1 日付** ― `js/main.js` の `formatDate` が `() => "YYYY.MM.DD"`（渡した日付を使わず、固定の文字を返している）。`index.html` のお知らせにも `YYYY.MM.DD` が直接書かれている。公開サイトでも5件すべてこの表示。
- **🔴2 FAQ欠落** ― 実装ページは index / news / medical / about / access の5枚。制作依頼書 4-1 は6ページ指定（「よくあるご質問」を含む）で、掲載原稿 §4 にFAQ本文がある。ナビにも公開サイトにもFAQは無い。
- **🔴3 heroコピー** ― `index.html` のヒーロー `<h1>` が独自の文言。制作依頼書 §5「キャッチコピー：モットー『地域に根ざした医療』を目立たせる」と照合。
- **🟡4 ヘッダー横あふれ** ― `css/style.css` の `.header-inner` が `width: var(--content-width)`（＝1180px固定）。`@media (max-width:1199px)` でも幅を戻していないため、821〜1180px でヘッダーが画面からはみ出す。フッター `.footer-inner` は `min()` で組めている（＝こちらが手本）。
- **🟡5 時間表の記号** ― index / access の診療時間表が `○ / ×`。掲載原稿は `● / ／` で凡例「●＝診療 ／＝休診」付き。実装には凡例が無い。
- **🟡6 電話リンク** ― `index.html` のアクセス欄の電話は素のテキスト。`access.html` は `tel:` リンクになっている（＝トップも同じにできる）。
- **🟡7 原稿の要約・欠落** ― `about.html` の理念が6項目（掲載原稿 §2 は7項目・「ライフステージに応じた医療」が抜けている）。診療内容・当院についての本文も短く要約。掲載原稿「文言改変不可・追加は要連絡」と照合。
- **💭9 スタッフ写真** ― `css/style.css` の `.staff-photo img` が `object-fit: fill` ＋ 各写真ごとに `top / left / width / height` を % のマジックナンバーで指定。
